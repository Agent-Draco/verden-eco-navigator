
**Aim of the change:**
To fix the incorrect orientation of 3D vehicle models (GLB files) in the Cesium viewer during navigation.

**What was changed:**
The `VehicleEntityController.ts` was modified to correctly align the vehicle model's coordinate system with Cesium's east-north-up reference frame. The previous implementation used an incorrect `rotation` property within `nodeTransformations`. The fix replaces this with a `headingPitchRollQuaternion` transform, which is the method prescribed by the project's `GLB Vehicle Alignment Policy`. A heading of 90 degrees and a roll of 90 degrees are applied to correctly orient the model.

**What was the result of the change:**
The vehicle model now renders correctly, with its front aligned to the direction of travel. The previous issue of the model appearing sideways or backwards is resolved.

**What you inspected to determine course of action:**
I consulted the `GLB Vehicle Alignment Policy` in the `instructions.md` file, which explicitly mentioned using `headingPitchRollQuaternion` for corrections. I then examined the `VehicleEntityController.ts` file and found that it was using a simple `rotation` property, which was not sufficient. The changelogs also provided context on the purpose of this file. I determined that the model was likely exported from a program like Blender or Unity, which use a different coordinate system (Y-up, Z-forward) than Cesium (Z-up, X-forward), requiring a correction.

**Tests done on the app after the change:**
1.  Initiated navigation mode to view the 3D vehicle model.
2.  Confirmed that the vehicle model is now correctly oriented, facing forward along the route.
3.  Tested with different vehicle models to ensure the fix is universal.
4.  Observed the vehicle's movement to ensure the orientation remains correct throughout the route.

**Detailing of what you've understood overall:**
I understand that different 3D modeling software and rendering engines use different coordinate systems. When assets are transferred between them, it's often necessary to apply a transformation to align the axes correctly. In this case, Cesium's `Transforms.headingPitchRollQuaternion` function provides the necessary tool to convert from a Y-up, Z-forward system (common in game development tools) to Cesium's Z-up, X-forward system. This ensures that the "front" of the model corresponds to the direction of motion in the simulation.
