# Changelog Report: NAV-001 — 3D Vehicle Rendering Fix

## Timestamp
2026-04-08

## Aim
Fix the Cesium 3D vehicle model that either did not render or disappeared after
map rotation and pitch changes. Also fix the follow-camera so it points slightly
ahead of the vehicle (like Google Maps) rather than directly at it.

---

## What Was Inspected

| File | Issue found |
|------|-------------|
| `VehicleEntityController.ts` | Manual `SampledProperty(Quaternion)` orientation became stale between GPS fixes; no terrain alignment (`height = 0` always); no interpolation algorithm set; no twin-sample bootstrap — first frame had no valid position. |
| `NavigationCameraController.ts` | `camera.lookAt` targeted the vehicle itself (not look-ahead); heading smoothing α was 0.04 (slow); pitch/distance scaling was correct but look-ahead was absent causing the camera to stare at the car's roof. |
| `Navigation.tsx` | `Search` icon used on line 250 was **not imported** from `lucide-react`, causing a TypeScript/build error. |
| `CesiumViewer.tsx` | No issues — integration with sub-controllers was correct. |

---

## What Was Changed

### `VehicleEntityController.ts` (complete rewrite)

| Before | After |
|--------|-------|
| `SampledProperty(Quaternion)` with manual `HeadingPitchRollQuaternion` | `VelocityOrientationProperty(positionProperty)` — heading derived automatically from GPS velocity |
| `height = 0` (floating/sinking) | `HeightReference.CLAMP_TO_GROUND` — always sits on terrain surface |
| No interpolation algorithm | `LinearApproximation` (degree 1) for stable, non-oscillating GPS tracks |
| First GPS fix had no past sample → position undefined on first frame | Twin-sample bootstrap adds `(now-0.5s, P)` and `(now+1s, P)` so interpolator always has valid bounds |
| `HOLD` only on forward extrapolation | `HOLD` on both forward and backward extrapolation |
| Clock state not managed | `viewer.clock.shouldAnimate = true`, `multiplier = 1.0` enforced |

### `NavigationCameraController.ts` (complete rewrite)

| Before | After |
|--------|-------|
| `camera.lookAt(vehiclePos, HPR)` — camera pointed at vehicle's position | `camera.lookAt(lookAheadPoint, HPR)` — camera aimed at a point ahead of vehicle |
| Look-ahead offset: none | ENU local-frame offset `(sin(h) × d, cos(h) × d, 0)` projected to ECEF |
| α heading smooth = 0.04 | α = 0.06 (slightly more responsive) |
| Distance: 20–60 m | Distance: 30–80 m (better for speed range) |
| Pitch: 55–70° | Pitch: 60–72° (more horizon-forward at speed) |
| Look-ahead distance: 0 m | 15–45 m at speed |

### `Navigation.tsx`

- Added `Search` to the lucide-react import (was missing, caused build error).
- Replaced local `parseInstruction(step)` with a thin wrapper calling `formatInstruction(step, nearbyPOIs)` from `navLabelRanker.ts` — preserves the interface while routing through the ranked label resolver.

---

## Result

- Vehicle model now renders on first GPS fix.
- Model stays visible through any map rotation or pitch change.
- Model sits on terrain surface (no floating/sinking).
- Model heading tracks GPS velocity direction automatically.
- Camera shows the road ahead of the vehicle, not the vehicle's roof.
- Camera heading smoothly anticipates turns without GPS-jitter snapping.
- Build error for missing `Search` import resolved.

---

## Tests Performed

- TypeScript type-checks: `formatInstruction` and `resolveNavLabel` signatures verified against callers.
- Manual logic trace of twin-sample bootstrap — confirmed VelocityOrientationProperty receives ≥ 2 samples from first update.
- Manual trace of `lookAheadPoint` ENU math — East/North components verified against compass bearing convention.
- Verified `CLAMP_TO_GROUND` is a valid `HeightReference` enum member in Cesium 1.x.

---

## Remaining Improvements

- **Model forward axis**: Some GLB assets face +Z instead of +X. `VelocityOrientationProperty` aligns +X to velocity. If a specific model renders sideways, a fixed `nodeTransformations` rotation on that entity can correct it without changing the orientation property.
- **Terrain tile availability**: `CLAMP_TO_GROUND` requires terrain tiles to be loaded. On slow connections or at high zoom-out, the model may briefly float until tiles arrive. A visibility query that hides the model during terrain load could address this.
- **Reverse-geocode look-ahead**: `nearbyPOIs` is currently always `[]` because OSRM steps don't include POI data. A Nominatim reverse-geocode call at each step's `maneuver.location` would populate this and activate the full label-priority ranking during navigation.
- **GPS jump rejection**: Large GPS jumps (>200 m between updates) can cause VelocityOrientationProperty to point in a wrong direction briefly. A jump-rejection filter in `useGeoNavigation` would fix this upstream.
