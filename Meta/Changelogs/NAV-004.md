# Changelog Report [NAV-004]

## Aim
The goal was to restore navbar visibility on the home screen map while keeping it hidden during active navigation, and to fix the missing route polyline in the 3D Cesium view.

## Changes
- **Navbar Visibility**:
    - Implemented a `useEffect` in `Home.tsx` to automatically sync `isNavHidden` with the `isOverlayOpen` state. This ensures the navbar returns as soon as search is exited.
    - Updated `BottomNav.tsx` to hide the navbar on `/navigation-dev`, matching the main `/navigation` screen behavior.
- **Polyline Rendering**:
    - Increased vertical offset to `0.5m` above terrain to prevent z-fighting.
    - Simplified `depthFailMaterial` to a basic `Color` with alpha for maximum hardware/driver compatibility.
    - Improved terrain sampling fallback logic in `CesiumViewer.tsx`.

## Result
Users can now browse the map with the full navbar available, and only see it disappear when they enter the focused "Where to?" search mode. In navigation mode, the route is now clearly visible even through terrain.

## Inspection & Research
- Verified that `isNavHidden` was being set but not unset.
- Researched Cesium `depthFailMaterial` limitations regarding `MaterialProperty` vs basic colors.

## Testing Done
- **UI Test**: Verified navbar state transitions in the local dev environment.
- **Rendering Test**: Confirmed polyline visibility on simulated terrain.
